
# VULNERABILITY MANAGEMENT POLICY

| **Owner** | Security Department | **Last Updated** | November 7, 2025 |
|---|---|---|---|

---

## 1. Purpose

The purpose of this Vulnerability Management Policy is to establish a structured and consistent process for the proactive identification, assessment, remediation, and reporting of security weaknesses across all organizational assets.

---

## 2. Scope

This policy applies to all technology assets owned, leased, or managed by the organization, including, but not limited to: servers, network devices, workstations, cloud environments, internally developed applications, and third-party software used by employees or contractors.

---

## 3. Roles and Responsibilities

| Role/Department | Responsibilities |
|---|---|
| **Security Team** | Leads the program, defines scanning schedules, analyzes all vulnerability data, and manages the exception process. |
| **IT Operations** | Responsible for applying patches and configuration changes to address identified vulnerabilities within established service level agreements (SLAs). |
| **Application Owners** | Responsible for remediation of vulnerabilities within their dedicated applications, particularly those found in custom code or dependencies. |
| **Management** | Responsible for approving policy, allocating resources, and reviewing periodic risk and remediation reports. |

---

## 4. Detection
- Vulnerability Scanning (VUM-01 - Monthly - RA-01) - The Mark will monthly conduct monthly vulnerabiltiy scans
- Penetration Test (VUM-02 - Yearly - RA-01) - The Dan will conduct annual penetration test
- Operation System Review (VUM-03 - Yearly - RA-01) - Configuration(OS/software) Review

----------

## 5. Configuration
- Configuration Standard (VUM-04 - Static - RA-01) - The Jonah will define a configuration standard
- Automated Configuration enforcement (VUM-05 - Weekly - RA-01) - All systems should have a software that enforce configs
- Only necessary software (VUM-06 - Daily - RA-01) - Systems will be deployed with only the necessary software

----------

## 6. Remediation
- Patching (VUM-07 - Daily - RA-01) - Systems will be patched monthly
- System Remediation (VUM-08 - Static - RA-01) - System will be remediated in line with the vulnerability management policy
- Remediation Scans (VUM-09 - Daily - RA-01) - Scans resulting in a failure will require a secondary scan to verify remediation

----------

## 7. AV
- AV Required (VUM-10 - Quarterly - RA-01) - All systems should have AV
- IDs functionality is required (VUM-11 - Daily - RA-01) - Cloud platforms should have IDS enabled 
- Tamper Protection (VUM-12 - Static - RA-01) - AV must have tamper protection enabled
